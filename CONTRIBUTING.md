# Contributing to AWS-EKS-DevOps-Automation

Thank you for considering contributing! Please follow these guidelines to help maintain the quality of the project.

## How to Contribute
1. Fork the repository and create a new branch.
2. Make your changes and test them locally.
3. Submit a Pull Request with a clear description of the changes.

## Code Guidelines
- Follow best DevOps practices.
- Ensure all YAML/Helm/Terraform files are formatted correctly.
- Keep commits small and meaningful.

## Reporting Issues
- Use the issue template to report bugs.
- Include relevant logs, steps to reproduce, and expected behavior.
